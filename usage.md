# Rime 使用速查

本文记录当前配置中实际启用的输入方案、快捷键和输入技巧。

## 当前输入方案

方案选单中启用四个方案：

| 方案 | 默认输出 | 适用场景 |
| --- | --- | --- |
| 中文 | 雾凇拼音、简体、半角 | 日常中文和中英混输，也是默认方案 |
| 日语 | 平假名及汉字转换、半角英数字 | 普通日语文章 |
| 日语全角 | 全角片假名、全角英数字和符号 | 要求所有字符默认全角的表单 |
| 日语半角 | 半角片假名、半角英数字和符号 | 要求所有字符默认半角的旧系统或表单 |

旧 Jaroomaji、双拼和九宫格的配置文件仍然保留，但目前没有加入方案选单。

Rime 的方案选单本身是平铺菜单，不是真正的二级菜单。“日语全角”和
“日语半角”各自只有一个宽度开关，因此按 `Ctrl+.` 时恰好显示四个方案和
一个开关；“中文”仍保留原有的简繁、Emoji、标点等开关，项目较多时会分页。

## 常用快捷键

| 输入或快捷键 | 功能 |
| --- | --- |
| `Ctrl+.` | 打开当前方案的方案与字符状态菜单 |
| `F4` | 打开同一个方案和功能开关菜单，作为备用入口 |
| `[`、`]` | 候选上一页、下一页 |
| `Ctrl+[`、`Ctrl+]` | 上屏当前候选的第一个字、最后一个字 |
| `Tab`、`Shift+Tab` | 在各段拼音之间向后、向前移动光标 |
| `Ctrl+Shift+3` | 切换中英文标点 |
| `Ctrl+Shift+4` | 切换简体、繁体 |
| 左 `Shift` | 有输入码时，上屏原始输入码并切换到英文 |
| 右 `Shift` | 有输入码时，上屏当前候选文字并切换到英文 |
| `Enter` | 直接上屏输入码，不切换输入状态 |

`Caps Lock` 保持普通的大写锁定用途，不负责切换中英文。`Shift+Space`
已被禁用。

### 编辑快捷键

组合输入过程中支持一组 Emacs 风格快捷键：

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl+P`、`Ctrl+N` | 候选上移、下移 |
| `Ctrl+B`、`Ctrl+F` | 光标左移、右移 |
| `Ctrl+A`、`Ctrl+E` | 移到输入码开头、结尾 |
| `Ctrl+D` | 向后删除 |
| `Ctrl+H` | 退格 |
| `Ctrl+K` | 删除光标后的内容 |
| `Ctrl+G` | 取消当前输入 |

小键盘数字及小数点在组合输入时会映射为普通数字和小数点。

## 日期、时间、数字与 Unicode

| 输入 | 功能 |
| --- | --- |
| `rq` | 当前日期，提供 `YYYY-MM-DD`、斜线、点号、紧凑格式和中文格式 |
| `rqzh` | 中文日期 |
| `rqen` | 英式和美式英文日期 |
| `sj` | 当前时间，提供时分和时分秒 |
| `xq` | 当前星期，提供“星期”“礼拜”“周”三种形式 |
| `dt` | ISO 8601 时间和紧凑日期时间 |
| `ts` | Unix 秒级时间戳 |
| `nl` | 当前农历日期 |
| `N20260723` | 将指定公历日期转换为农历 |
| `uuid` | 生成 UUID v4 |
| `cC1+2*3` | 内置计算器 |
| `U62fc` | 按 Unicode 码位输入字符，例如“拼” |
| `R1234.56` | 数字小写、数字大写、金额小写和金额大写 |

数字转换也可以使用其他大写字母引导，但建议统一使用 `R`；`U` 已由
Unicode 输入占用。

## 中文输入

雾凇拼音默认使用中文、简体、半角模式，并开启 Emoji。每页显示 5 个候选。

当前开启的拼音功能包括：

- 超级简拼，支持使用声母快速输入长词。
- `zh`、`ch`、`sh` 在超级简拼中作为整体处理。
- 常见按键顺序错误自动纠正，例如 `hzi` 可以匹配 `zhi`。
- 常见韵母错序自动纠正。
- 兼容 `nue/nve`、`qu/qv` 等拼法。
- 对部分错音和错字在候选注释中给出提示。
- 中文、英文和中英混合词共同参与候选。
- 降低容易与拼音冲突的短英文单词的候选位置。
- 长词优先：从靠后的候选中提升少量更长的中文词。
- 对个人常用候选进行置顶，但仍保留用户词频学习和造词能力。
- 对四音节以上的长词提供自动补全（需要较新的 librime）。

词库还允许大写字母和数字参与中文造词，例如 `5G网络`、`3D打印` 和
`M4A1`。

### 英文大小写

英文自动大小写过滤器已经开启：

- 输入码首字母大写时，英文候选转换为首字母大写。
- 输入码前两个或更多字母大写时，英文候选转换为全大写。

英文翻译器不造句、不学习词频，以免干扰中文用户词典。

## 自定义短语

`custom_phrase.txt` 只保存需要自创编码的固定短语，并具有很高的候选优先级。
目前主要包括：

- 常用缩写，例如 `bzd` 对应“不知道”，`wsm` 对应“为什么”。
- 叠词和口语，例如 `dd` 对应“等等”，`hh` 对应“哈哈”。

自定义短语不会和普通翻译器互相造词，因此适合固定简码和专有名词。

单字候选顺序现在由 `rime_ice.custom.yaml` 的候选置顶过滤器管理，例如
`w` 对应“我、万、往”，`d` 对应“的、地、得”。这种方式只调整已有候选
顺序，不会阻止学习和造词。

个人词库分为：

- `cn_dicts/personal.dict.yaml`：个人中文和中英混合词。
- `en_dicts/personal_en.dict.yaml`：个人英文技术词库，按领域分类并带权重；
  其中长词支持前缀补全，也可使用 `chan`（ChannelAccess）、`aread`
  （areaDetector）、`systemv`（SystemVerilog）等确定性缩写；涵盖 EPICS、
  Linux、Raspberry Pi、VME、MicroTCA、PCIe、FPGA、Python、Git 等主题。
- `kagiroi_custom_phrases.txt`：个人日语人名、专有名词和固定读法，格式为
  `词语<Tab>平假名读音`。

## `v` 特殊符号模式

全拼方案使用 `v` 作为特殊符号前缀。常用编码如下：

| 输入 | 内容 |
| --- | --- |
| `va` | `ā á ǎ à` 等带音调字母 |
| `v1`～`v10` | 中文数字、罗马数字、圈号数字等 |
| `vfs` | 分数 |
| `vjt` | 箭头 |
| `vsx` | 数学符号 |
| `vdw` | 单位符号 |
| `vhb` | 货币符号 |
| `vxl`、`vxld` | 小写、大写希腊字母 |
| `vpy` | 拼音声调字符 |
| `vzy` | 注音符号 |
| `vjm`、`vpjm` | 平假名、片假名 |
| `vbq` | 表情符号 |
| `vtq` | 天气符号 |
| `vyy` | 音乐符号 |
| `vxz` | 星座 |
| `vbd` | 标点 |
| `vdongwu` | 动物 Emoji |
| `vtianqi` | 天气 Emoji |
| `vgongju` | 工具 Emoji |
| `vxinxing` | 彩色心形 Emoji |

输入两个字符的 `v` 编码时，单字符符号会被提升到英文候选之前。例如输入
`va` 时，带声调的字母会排在 `van` 等英文候选前面。

## Emoji

Emoji 转换默认开启。正常输入中文词时，相关 Emoji 会出现在候选中。

还提供以下批量 Emoji 入口：

- `V动物`
- `V国旗`
- `V手势`
- `V表情`
- `V分数`
- `V按键`
- `V星号`
- `V特殊`
- `V常用`
- `V中华`
- `V中餐`

也可以使用 `v动物`、`v天气` 等分类符号编码。

## 部件拼字反查

使用 `uU` 开头进入部件拼字反查，用组成部件查找不会读或不会写的汉字。
双字母前缀可以避免截获 `user`、`update`、`ubuntu` 等英文。反查结果会显示
本方案中的拼音，还可以使用反引号继续输入辅码筛选。

## 日语输入

当前日语方案使用 Kagiroi。它以 Mozc 词典和连接矩阵为基础，通过 Viterbi
算法生成词语及整句候选，并开启用户词频学习。

Kagiroi 需要 librime 1.11.2 或更高版本以及 librime-lua。由于词典和连接
矩阵较大，首次重新部署会明显慢于原日语方案，之后正常输入不需要重复编译。

- 罗马字输入会先转换为平假名，再参与假名到汉字的整句转换。
- `-` 或 `q` 输入长音符 `ー`。
- `nn` 输入 `ん`；输入末尾单独的 `n` 也会按 `ん` 参与转换。
- 支持双写辅音、小写假名和拗音。
- 同时支持 `shi/si`、`chi/ti`、`tsu/tu` 等常见拼法。
- 支持按词和整句选词；常用结果会通过同一个 Kagiroi 用户词典学习。
- 三个日语方案共用 Kagiroi 词典、转换模型、个人固定词条和用户学习记录。
- Emoji 转换默认开启，但不再占用日语方案菜单。

个人固定词条写入 `kagiroi_custom_phrases.txt`，每行使用
`词语<Tab>平假名读音`。这个文件与上游词库分开，今后更新 Kagiroi 时不会
被覆盖。例如：

```text
固有名詞	こゆうめいし
```

Kagiroi 还支持义训（Gikun）：选定前一段文字后，以 `;` 分隔读音和想要
关联的文字，可将非标准读法记入用户词典。

### 三个日语方案

“日语”是标准方案。罗马字先转换为平假名，再提供假名、汉字和整句候选；
通过 Shift 切换到英文时使用半角英数字。它没有额外菜单项，适合绝大多数
普通日语输入。

“日语全角”固定使用全角片假名：

- `yamada` 输出 `ヤマダ`。
- 英文字母、数字、空格和符号默认为全角，例如 `ＡＢＣ１２３＠－＿`。
- `Ctrl+.` 中选择“英数符号半角”后，英数字和符号改为半角，但假名仍是
  全角片假名。

“日语半角”固定使用半角片假名：

- `yamada` 输出 `ﾔﾏﾀﾞ`，浊点、半浊点和长音也使用半角形式。
- 英文字母、数字、空格和符号默认为半角。
- `Ctrl+.` 中选择“英数符号全角”后，英数字和符号改为全角，但假名仍是
  半角片假名。

三个方案中，Shift 仍负责切换日语转换和直接英文输入；在全角/半角专用
方案里，Shift 英文遵循当前的“英数符号半角 / 全角”状态。

标准日语的常用标点：

- `,` 输出 `、`
- `.` 输出 `。`
- `/` 可选 `・`、`…`、`，`、`／`
- 方括号可选择日语引号

半角符号状态下，`,`、`.`、`/` 和方括号分别提供
`､`、`｡`、`･`、`｢｣`，其余常用 ASCII 符号保持半角。

更详细的罗马字转换规则参见
[romaji-to-kana.md](romaji-to-kana.md)。

## 特定应用默认英文

macOS 中以下应用默认进入英文模式：

- Spotlight
- Alacritty
- MacDown
- Raycast
- Visual Studio Code

Windows 中以下应用默认进入英文模式：

- CMD 和 Conhost
- Everything
- IntelliJ IDEA
- PyCharm
- Photoshop

## 当前没有启用的功能

下列配置或文件仍在项目中，但没有接入当前方案：

- 所有双拼方案和九宫格方案。
- 旧 `jaroomaji` 日语方案及其词库；需要回退时可重新加入方案列表。
- `41448` 生僻字大字表。
- `cn_dicts/biaoqing.dict.yaml`、`cn_dicts/emoji.dict.yaml` 和
  `cn_dicts/sogou.dict.yaml`。
- 中英混合词自动插入空格。
- 英文上屏后自动添加空格。
- 用户词典星号标记。
- 冷词隐藏、降频和强制删词。
